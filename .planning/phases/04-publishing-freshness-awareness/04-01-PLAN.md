<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| rotation state → publish projection | File-backed orchestration freshness crosses into live frontend artifacts |
| exporter → docs bundle | Generated JSON becomes user-visible static content |
| baseline artifact → merged snapshot/index | Older persisted data is reused in new live payloads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-04-01 | T | `publish_projection.py` freshness derivation | mitigate | Derive freshness only from `freshness[symbol].last_succeeded_at`; never use bundle-level `last_updated` for stock freshness |
| T-04-02 | I | `data.json` / `stock_index.json` | mitigate | Publish explicit freshness labels and `last_succeeded_at` so stale data is disclosed, not hidden |
| T-04-03 | T | `publish_safety.py` artifact validation | mitigate | Add `stock_index` artifact-kind validation and tests so malformed index payloads cannot be published |
| T-04-04 | D | `export_canslim.py` publish bundle | mitigate | Fail closed if baseline/projection payload validation fails before live docs replacement |
</threat_model>
