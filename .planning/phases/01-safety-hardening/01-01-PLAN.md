<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| script→filesystem | Untrusted or partial payloads cross into published GitHub Pages artifacts |
| concurrent writer→shared artifact | Multiple local/GitHub Actions processes compete for the same `docs/*.json` targets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T | `publish_safety.py` | mitigate | Validate payload shape before publish and refuse writes missing `schema_version`, `last_updated`, or `stocks`. |
| T-01-02 | D | `publish_safety.py` lock path | mitigate | Use per-target lock files with bounded timeout so concurrent writers serialize instead of corrupting artifacts. |
| T-01-03 | R | backup/restore flow | mitigate | Emit structured exceptions and return backup metadata so restore actions are logged and auditable. |
| T-01-04 | I | backup snapshots | accept | Backups remain local repo artifacts with no new secret material; phase scope does not add external transport. |
</threat_model>
