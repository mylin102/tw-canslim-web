## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T | `publish_safety.py` | mitigate | Validate payload shape before publish and refuse writes missing `schema_version`, `last_updated`, or `stocks`. |
 | T-01-02 | D | `publish_safety.py` lock path | mitigate | Use one bundle transaction lock with bounded timeout so related artifact publishes serialize instead of corrupting artifacts. |
| T-01-03 | R | backup/restore flow | mitigate | Emit structured exceptions and return backup metadata so restore actions are logged and auditable. |
| T-01-04 | I | backup snapshots | accept | Backups remain local repo artifacts with no new secret material; phase scope does not add external transport. |
